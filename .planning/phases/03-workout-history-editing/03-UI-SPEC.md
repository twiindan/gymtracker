## Spacing Scale

Tailwind v4 default spacing scale (all multiples of 4px). Confirmed from codebase usage patterns:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon button padding, inline badge gaps |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing, table cell padding |
| md | 16px (`p-4`, `gap-4`) | Default card padding, section gaps |
| lg | 24px (`p-6`, `gap-6`, `mb-6`) | Modal padding, exercise card gaps |
| xl | 32px (`mb-8`, `px-8`) | Major section breaks, page container padding |
| 2xl | 48px (`mb-12`) | Page-level spacing between major sections |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- `p-5` (20px) used on history list cards and filter panel — accepted as existing pattern
- `mb-10` (40px) used on history page header — accepted as existing pattern
- Touch targets: minimum 44px for interactive elements (buttons use `px-3 py-2` or `px-4 py-2` which exceed 44px on most fonts)

**Source:** Extracted from `workouts/page.tsx`, `workouts/[id]/page.tsx`, `set-input.tsx`, `exercise-picker.tsx`.

## Interaction Patterns (Locked from Existing Implementation)

### Edit Mode Toggle
- `isEditing` state toggles between read-only view and editable form
- Edit form appears inline at top of detail page (not modal, not full-page)
- Metadata fields: name (text input), date (datetime-local), notes (textarea)
- Save/Cancel buttons appear below metadata form
- Cancel resets form to original values

### Inline Set Editing
- Weight/Reps cells become `<input type="number">` when `isEditing` is true
- `onBlur` triggers immediate save to Supabase + refetch
- Weight: `step={0.5}`, `inputMode="decimal"`, `min={0}`
- Reps: `step={1}`, `inputMode="numeric"`, `min={0}`
- Non-editing mode shows plain text with "—" for null values

### Exercise Management
- "Add Exercise" button opens `ExercisePicker` modal (only visible in edit mode)
- Remove exercise: inline X button on exercise card header (only visible in edit mode)
- Add set: dashed border button below sets table (only visible in edit mode)
- Delete set: trash icon in table row (only visible in edit mode)

### Delete Confirmation
- Modal overlay: `fixed inset-0 z-50 flex items-center justify-center bg-black/50`
- Dialog card: `rounded-xl bg-white p-6 shadow-xl dark:bg-zinc-900`, `max-w-sm`
- Two-button layout: destructive primary (full-width flex-1) + neutral secondary

### History List
- Cards: `rounded-2xl border border-border bg-surface p-5`
- Hover: `hover:shadow-lg hover:shadow-primary/5 hover:border-primary/30 hover:-translate-y-0.5`
- Date badge: `h-12 w-12 rounded-xl bg-primary-light/50` with day number
- Each card links to `/workouts/{id}`

### Filters
- Collapsible filter panel: `rounded-2xl border border-border bg-surface p-5 shadow-sm`
- Filter pills: `rounded-full px-4 py-2 text-xs font-semibold`
- Active pill: `bg-primary text-white shadow-sm`
- Inactive pill: `bg-primary-light/50 text-primary hover:bg-primary-light`

### Modal/Overlay Patterns
- Exercise picker: `fixed inset-0 z-50 bg-black/60 backdrop-blur-sm`, bottom sheet on mobile (`items-end`), centered on desktop (`sm:items-center`)
- Custom exercise form: nested modal at `z-[60]`
- All modals: click outside to close (`e.target === e.currentTarget`)

### Loading States
- Spinner: `inline-flex h-8 w-8 animate-spin rounded-full border-2 border-primary border-t-transparent`
- Text: centered, `text-zinc-500` / `text-muted`
