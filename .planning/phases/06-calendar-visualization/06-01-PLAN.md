<task type="auto">
  <name>Task 1: Create calendar utility functions (data query, grid generation, color scale)</name>
  <files>src/lib/calendar-utils.ts</files>
  <action>
    Create `src/lib/calendar-utils.ts` with three exports:

    1. `generateCalendarGrid(days: number = 365)`:
       - Calculate endDate = new Date(), startDate = endDate - days
       - Align startDate to previous Sunday: `startDate.setDate(startDate.getDate() - startDate.getDay())`
       - Build weeks array: iterate day-by-day from startDate to endDate
       - Each day entry: `{ date: string (YYYY-MM-DD via toISOString().slice(0,10)), dayOfWeek: number }`
       - Push to currentWeek; when dayOfWeek === 6 (Saturday), push currentWeek to weeks and reset
       - Return `{ weeks, startDate, endDate }`
       - CRITICAL: Use `toISOString().slice(0, 10)` for consistent YYYY-MM-DD keys to avoid timezone off-by-one (Pitfall 1 from RESEARCH.md)

    2. `getHeatmapColor(count: number): string`:
       - 4-level discrete scale using CSS variable names (resolved via getComputedStyle or inline style):
         - count === 0 → `var(--border)` (empty/gray)
         - count === 1 → `var(--primary-light)` (light green)
         - count >= 2 && count <= 3 → `var(--primary)` (medium green)
         - count >= 4 → `var(--primary-dark)` (dark green)
       - Return the CSS variable string (SVG fill accepts CSS variables)

    3. `fetchWorkoutCalendarData(days: number = 365)`:
       - Create browser client: `const supabase = createBrowserClient()`
       - Calculate startDate: `new Date(); startDate.setDate(startDate.getDate() - days)`
       - Single aggregated query (NO N+1 — Pitfall 3 from RESEARCH.md):
         ```typescript
         const { data, error } = await supabase
           .from("workouts")
           .select("started_at")
           .is("deleted_at", null)  // Always filter soft-deleted workouts (Pitfall 2)
           .gte("started_at", startDate.toISOString());
         ```
       - Aggregate client-side: group by date string, count per day
         ```typescript
         const dateCounts = new Map<string, number>();
         (data ?? []).forEach((w) => {
           const dateStr = new Date(w.started_at).toISOString().slice(0, 10);
           dateCounts.set(dateStr, (dateCounts.get(dateStr) ?? 0) + 1);
         });
         ```
       - Return `dateCounts` as `Map<string, number>` (key = "YYYY-MM-DD", value = workout count)
       - If error, throw it

    DO NOT import date-fns or any external date library — use native Date methods only.
    DO NOT use react-calendar-heatmap or @uiw/react-heat-map (CSS import issues with Next.js).
  </action>
  <verify>
    <automated>npx tsc --noEmit src/lib/calendar-utils.ts</automated>
  </verify>
  <done>
    - src/lib/calendar-utils.ts exports generateCalendarGrid, getHeatmapColor, fetchWorkoutCalendarData
    - generateCalendarGrid returns weeks array aligned to Sunday start
    - getHeatmapColor returns 4-level CSS variable strings
    - fetchWorkoutCalendarData uses single query with .is("deleted_at", null) filter
