  <name>Task 1: Create muscle volume data fetching utility</name>
  <files>src/lib/muscle-volume-utils.ts</files>
  <action>
    Create `src/lib/muscle-volume-utils.ts` with:

    1. Type definition:
       ```typescript
       export interface MuscleVolumeData {
         muscle_group: string;
         total_sets: number;
         total_volume: number;  // sum of (weight * reps) for all sets
       }
       ```

    2. `fetchMuscleVolume(days: number = 30): Promise<MuscleVolumeData[]>`:
       - Create browser client: `const supabase = createBrowserClient()`
       - Calculate startDate: `new Date(); startDate.setDate(startDate.getDate() - days)`

       - Step 1: Get non-deleted workout IDs in time window (single query):
         ```typescript
         const { data: workouts } = await supabase
           .from("workouts")
           .select("id")
           .is("deleted_at", null)  // Always filter soft-deleted workouts
           .gte("started_at", startDate.toISOString());
         ```
         If no workouts, return `[]`.

       - Step 2: Get workout_exercises with primary_muscle_group (denormalized — no need to join exercises table):
         ```typescript
         const workoutIds = workouts.map((w) => w.id);
         const { data: weData } = await supabase
           .from("workout_exercises")
           .select("id, primary_muscle_group")
           .in("workout_id", workoutIds);
         ```
         If no workout_exercises, return `[]`.

       - Step 3: Get all sets for these workout_exercises and aggregate:
         ```typescript
         const weIds = weData.map((we) => we.id);
         const weMap = new Map(weData.map((we) => [we.id, we.primary_muscle_group]));

         const { data: sets } = await supabase
           .from("sets")
           .select("workout_exercise_id, reps, weight")
           .in("workout_exercise_id", weIds);
         ```

       - Step 4: Client-side aggregation by muscle_group:
         ```typescript
         const muscleMap = new Map<string, { total_sets: number; total_volume: number }>();
         (sets ?? []).forEach((set) => {
           const muscle = weMap.get(set.workout_exercise_id);
           if (!muscle) return;
           const entry = muscleMap.get(muscle) ?? { total_sets: 0, total_volume: 0 };
           entry.total_sets += 1;
           if (set.reps != null && set.weight != null) {
             entry.total_volume += set.reps * set.weight;
           }
           muscleMap.set(muscle, entry);
         });
         ```

       - Step 5: Convert to array, sort by total_sets descending, return:
         ```typescript
         return Array.from(muscleMap.entries())
           .map(([muscle_group, data]) => ({ muscle_group, ...data }))
           .sort((a, b) => b.total_sets - a.total_sets);
         ```

       NOTE: This uses 3 sequential queries (workouts → workout_exercises → sets) which is acceptable for Supabase client-side. The alternative would be an RPC function, but that's overkill for this phase. The key is NOT doing N+1 (one query per workout).

       CRITICAL: Count ONLY primary_muscle_group from workout_exercises. Do NOT count secondary_muscle_groups (deferred to v2.x per CONTEXT.md).

       CRITICAL: Always include `.is("deleted_at", null)` on the workouts query.
  </action>
