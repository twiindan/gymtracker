<objective>
Add auto-progression suggestions to the active workout page. When a user starts a workout, the system analyzes their previous session history and displays progression suggestions for exercises where they hit a PR or plateaued (same weight/reps for 2+ consecutive sessions per D-68). Suggestions use linear progression: +2.5kg upper body, +1.25kg lower body, or +1 rep (per D-66). Users can accept, reject, or modify suggestions (per D-69).

Purpose: Provide intelligent, contextual guidance at the moment of logging — suggestions appear on the active workout page right when the user is about to log (per D-67).
Output: Progression utility library, suggestion panel component integrated into active workout page.
</objective>

Create `src/lib/progression-utils.ts` with the following exports:

1. **`UPPER_BODY_MUSCLES` constant** — array of muscle group strings classified as upper body:
   ```typescript
   export const UPPER_BODY_MUSCLES = [
     "Chest", "Back", "Shoulders", "Biceps", "Triceps",
     "Forearms", "Traps", "Lats", "Abs",
   ];
   ```
   Lower body muscles are everything else in `MUSCLE_GROUPS` from schema.ts: "Quadriceps", "Hamstrings", "Glutes", "Calves", "Lower Back", "Full Body", "Cardio".

2. **`isUpperBodyMuscle(muscleGroup: string): boolean`** — returns true if the muscle group is in UPPER_BODY_MUSCLES.

3. **`detectPlateau(sets: Array<{ weight: number | null; reps: number | null }>, minConsecutive: number): boolean`** — detects if the last `minConsecutive` sets (from different sessions) have the same weight and reps. Returns true if plateau detected. For this implementation, `minConsecutive = 2` (per D-68).

4. **`getProgressionSuggestion(exerciseId: string, exerciseName: string, primaryMuscleGroup: string, previousWeight: number | null, previousReps: number | null, hasPlateau: boolean, hasPR: boolean): ProgressionSuggestion | null`** — returns a suggestion object or null:
   - If neither hasPR nor hasPlateau: return null
   - If hasPR or hasPlateau:
     - If previousWeight is not null: calculate suggested weight increment
       - Upper body: `suggestedWeight = previousWeight + 2.5`
       - Lower body: `suggestedWeight = previousWeight + 1.25`
       - `suggestedReps = previousReps`
     - If previousWeight is null (bodyweight exercise): `suggestedWeight = null`, `suggestedReps = (previousReps ?? 0) + 1`
   - Return: `{ exerciseId, exerciseName, primaryMuscleGroup, previousWeight, previousReps, suggestedWeight, suggestedReps, reason: hasPR ? "pr" : "plateau" }`

5. **`fetchExerciseHistory(supabase, exerciseId: string, limit: number): Promise<Array<{ weight: number | null; reps: number | null; workout_started_at: string }>>`** — queries the sets table joined with workout_exercises and workouts to get the most recent completed sets for an exercise, ordered by workout date descending. Returns up to `limit` most recent set entries.

   Query pattern:
   ```typescript
   const { data } = await supabase
     .from("sets")
     .select("weight, reps, workout_exercises!inner(workout_id, workouts!inner(started_at))")
     .eq("workout_exercises.exercise_id", exerciseId)
     .order("workouts.started_at", { ascending: false })
     .limit(limit);
   ```

   Flatten the nested structure to return `{ weight, reps, workout_started_at }[]`.

     <div className="mb-4 rounded-xl border border-zinc-200 bg-white p-4 dark:border-zinc-800 dark:bg-zinc-900">
       <div className="text-sm text-zinc-500">Loading suggestions...</div>

               if (!prev) return prev;
               return {

                     sets: ex.sets.map((s, i) =>

                     ),
                   };
                 }),
               };
             });

             <div className="font-semibold">{suggestion.exerciseName}</div>
