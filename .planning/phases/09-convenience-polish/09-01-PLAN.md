**1. `fetchAllExportData()`** — Fetches ALL user data from Supabase in parallel using `Promise.all`. Per D-84, includes:
- `exercises`: All custom exercises (`is_custom: true`) from `exercises` table — select: id, name, primary_muscle_group, secondary_muscle_groups, equipment, tracking_type, description, created_at
- `workouts`: All non-deleted workouts (`deleted_at: is.null`) from `workouts` table — select: id, name, routine_id, scheduled_workout_id, started_at, ended_at, notes, created_at. Order by `started_at` ascending.
- `workout_exercises`: All entries from `workout_exercises` table — select: id, workout_id, exercise_id, exercise_name, primary_muscle_group, sort_order, notes
- `sets`: All entries from `sets` table — select: id, workout_exercise_id, set_number, reps, weight, rpe, notes, created_at
- `routines`: All from `routines` table — select: id, name, description, folder_id, created_at, updated_at
- `routine_exercises`: All from `routine_exercises` table — select: id, routine_id, exercise_id, exercise_name, primary_muscle_group, sort_order, target_sets, target_reps_min, target_reps_max, notes
- `scheduled_workouts`: All from `scheduled_workouts` table — select: id, name, scheduled_date, routine_id, notes, created_at
- `routine_folders`: All from `routine_folders` table — select: id, name, parent_id, created_at

Use `createBrowserClient()` for all queries. Return type:
```typescript
export interface ExportData {
  exercises: Exercise[];
```

**2. `buildJsonExport(data: ExportData): string`** — Per D-88, produces a single JSON string with nested structure. Structure:
```json
{
  "exportMetadata": {
    "exportedAt": "2026-04-29T12:00:00.000Z",
    "appVersion": "GymTracker v2.0"
  },
  "workouts": [
    {
      "id": "...",
      "name": "...",
      "startedAt": "2026-04-29T10:00:00.000Z",
      "endedAt": "...",
      "notes": "...",
      "exercises": [
        {
          "name": "Bench Press",
          "primaryMuscleGroup": "Chest",
          "sortOrder": 0,
          "notes": "...",
          "sets": [
            { "setNumber": 1, "weight": 80, "reps": 8, "rpe": 7, "notes": "" }
          ]
        }
      ]
    }
  ],
  "routines": [
    {
      "id": "...",
      "name": "Push Day",
      "description": "...",
      "exercises": [
        { "name": "Bench Press", "primaryMuscleGroup": "Chest", "sortOrder": 0, "targetSets": 4, "targetRepsMin": 6, "targetRepsMax": 10, "notes": "" }
      ]
    }
  ],
  "scheduledWorkouts": [...],
  "routineFolders": [...],
  "customExercises": [...]
}
```

Key rules:
- Convert snake_case DB columns to camelCase for JSON keys (e.g., `started_at` → `startedAt`, `primary_muscle_group` → `primaryMuscleGroup`)
- Nest `sets` inside `workout_exercises` inside `workouts` (join on `workout_id` then `workout_exercise_id`)
- Nest `routine_exercises` inside `routines` (join on `routine_id`)
- Include `exportMetadata` with `exportedAt` (ISO 8601 via `new Date().toISOString()`) and `appVersion: "GymTracker v2.0"`
- Use `JSON.stringify(result, null, 2)` for pretty-printed output
- Only include custom exercises in `customExercises` array (filter `is_custom: true`)

**3. `buildCsvExport(data: ExportData): string`** — Per D-89, produces flat denormalized CSV with one row per set. Column headers (in this exact order):
```
workout_date,workout_name,exercise_name,set_number,weight,reps,rpe,notes,primary_muscle_group,tracking_type
```

Key rules:
- One row per set (iterate sets, join to workout_exercises by `workout_exercise_id`, join to workouts by `workout_id`)
- `workout_date` = extract date portion from `workout.started_at` (e.g., "2026-04-29")
- `weight`, `reps`, `rpe` = numeric values or empty string if null
- `notes` = escape commas by wrapping in double quotes if they contain commas; escape double quotes by doubling them
- Include a header row as the first line
- Use `\n` as line separator (no trailing newline after last row)
- If no sets exist, return just the header row

**4. `downloadFile(content: string, filename: string, mimeType: string): void`** — Triggers browser download:
- Create `Blob` with given content and mimeType
- Create object URL via `URL.createObjectURL(blob)`
- Create temporary `<a>` element with `href` = URL and `download` = filename
- Programmatically click the link
- Clean up: `URL.revokeObjectURL(url)` and remove the `<a>` element
- Filename format: `gymtracker-export-YYYY-MM-DD.json` or `gymtracker-export-YYYY-MM-DD.csv` (date from `new Date().toISOString().split('T')[0]`)
